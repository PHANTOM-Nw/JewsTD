# 游戏配置入口

运行时数值以 `src/game/config/` 为唯一事实来源。修改数值时直接编辑对应 TypeScript 文件并更新测试，不在文档维护第二份完整配置表。

| 文件 | 当前职责 | 典型修改 |
| --- | --- | --- |
| `mahjong.ts` | 牌池、牌面、隐藏视图、花色随机区间、面子倍率与机制、中发白参数 | 麻将牌流、战斗与合成平衡 |
| `economy.ts` | 起始资源、每轮建造次数、普通/纯墙拆除价格 | 回合资源与经济节奏 |
| `map.ts` | 网格尺寸、入口、出口和必经点 | 地图布局与基础路线 |
| `enemies.ts` | 敌人生命、速度、半径、抗性、奖励、漏怪伤害和交通间隙 | 敌人平衡与队列密度 |
| `waves.ts` | 12 波敌人组合、生成间隔和生命倍率 | 难度曲线与 Boss 波 |
| `towers.ts` | 旧宝石塔与特殊塔兼容配置 | 非当前麻将运行时；不要据此新增玩家入口 |

领域类型位于 `src/game/types/game.ts`。当前生产玩法使用 108 张万、条、筒实体数牌、5 种麻将形态、3 张功能牌、普通/纯两类墙、4 类敌人和连续 12 波。第 12 波必须保留 `isBossWave` 并包含 Boss 生成组。

## 当前关键配置边界

- 地图为 10 行×8 列，每格 40 逻辑像素。`WAYPOINTS` 是路线锚点的唯一坐标来源；入口 `(0,0)`、出口 `(9,7)`，空图路线共 16 步。
- 每轮新摸 5 张牌、成功放置 3 张、激活 1 张并产生 2 张普通牌墙。
- 开局 50 金币、15 点矿坑生命。普通牌墙拆除 100 金币并返还实体牌；纯墙体拆除 50 金币且不返牌。
- `economy.ts` 中的 `obstacleRemovalGoldCost: 20` 和 `maxObstacles: 24` 是旧障碍兼容配置；当前麻将 UI 不提供 20 金币通用清障，也不会调用 `recycleOldestObstacles` 自动风化麻将墙体。
- 花色随机属性、面子倍率、暴击/破甲、毒、减速/溅射以及中发白数值全部由 `mahjong.ts` 提供；`useGameEngine` 不应另设麻将数值副本。
- 12 波生命倍率从第 1 波 1.0 增长到第 12 波 7.0；最终波包含 1 个 Boss。具体敌人数量和间隔以 `waves.ts` 为准。
- 敌人的最小沿路间距由双方半径之和加交通间隙计算；半径还决定后兵是否足以推动处于减速且未眩晕的前兵。

旧 `towers.ts` 中的 `multiTarget`、`splashRadius`、`pierce`、旧毒素和眩晕字段仍由兼容战斗分支支持，但麻将棋子读取 `mahjong.ts` 并通过 `mahjongAttack.ts` 结算。修改旧宝石配置不会改变麻将塔的花色或面子能力。

## 修改检查表

1. 修改麻将牌池、隐藏视图或实体字段时，确认 108 张实体唯一且 UI 不泄露暗牌花色、点数、实体对象或随机属性。
2. 修改随机区间、面子倍率或中发白参数时，同步 `mahjongStats`、`mahjongAttack`、合成预览文案和对应测试。
3. 修改合成路线时，确认阶段权限、锚点、白/牌墙限制、材料位置转纯墙、实体归属和失败原子性。
4. 修改墙体价格或回池规则时，同时更新 `economy.ts`、墙体事务、UI 提示和牌池唯一性测试。
5. 修改地图尺寸或必经点时，确认坐标合法并更新空图路径、放置容量与 Pointer 坐标测试。
6. 修改敌人或波次时，维护数量、间隔、半径、交通、生命倍率、入口净空、推动关系和最终 Boss 约束。
7. 执行 `npm run lint`、`npm test` 和 `npm run build`；玩法变化同步当前文档并在 `docs/records/` 添加记录。
