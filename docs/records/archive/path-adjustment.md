## 🎯 路径设计特点

### S形/螺旋形路径结构

```
起点(绿色, 顶部 0,2)
  ↓ 向下
转折点1 (左侧)
  → 向右  
转折点2
  ↓ 向下到矿坑
**矿坑**(黄色中心, 7,10)
  → 向右
转折点3 (右侧)
  ↑ 向上
转折点4 (右上)
  ← 向左
转折点5 (中上)
  ↓ 向下
转折点6 (中下)
  → 向右
终点(红色,右下 14,19)
```

### 路径优势

1. **增加游戏难度**: 敌人需要走更长的路程,给玩家更多时间建造防御
2. **策略性更强**: 玩家可以利用多个转弯点布置塔
3. **视觉效果更好**: 复杂的路径让游戏更有挑战性
4. **符合原版设计**: 接近宝石TD原版的复杂路径设计

## 🚀 使用方法

1. **启动游戏**:
   ```bash
   npm run dev
   ```

2. **查看路径**:
   - 打开浏览器访问 http://localhost:5177/
   - 游戏初始化时会自动计算并显示路径
   - 控制台会输出详细的路径信息

3. **放置塔**:
   - 点击空地放置随机宝石塔
   - 注意不要堵死路径(系统会自动检测)

4. **开始波次**:
   - 点击"开始下一波"按钮
   - 敌人会沿着红色虚线路径移动

---

## ⚠️ 注意事项

1. **路径阻塞检测**: 
   - 放置塔时会使用BFS验证是否堵死路径
   - 如果会堵死,会弹出警告并阻止放置

2. **障碍物不可覆盖**:
   - 预定义障碍物不能被塔覆盖
   - 变成障碍物的塔也不能被移除

3. **路径实时更新**:
   - 每次放置塔或塔变成障碍物后,路径会重新计算
   - 可视化路径会同步更新
