# 中發白赌博概率重做

日期：2026-07-16

## 背景与动机

`resolving_hand` 阶段处理剩余牌的旧规则是一个单向门：当剩余恰为“1 张旧手牌 + 2 张新暗牌”时，玩家必须先在“看花色选牌”与“赌中发白”之间二选一——一旦选择赌博，两张新牌的花色始终不公开，只能盲赌；判定本身也是布尔式的，只有三张牌全同花色才成功且必然成功，花色不齐则必然失败。

这套设计有两个问题：其一，玩家在决定是否赌博时完全看不到任何花色信息，纯靠盲猜，没有任何工具可以降低不确定性带来的负反馈；其二，赌博结果由“是否全同花色”这一固定布尔判据决定，没有概率张力，花色不齐就注定拿不到功能牌，体验上更接近一次性抽卡失败而非博弈取舍，挫败感偏重。本次改动重做该规则，目标是保留“资源转化为功能牌”的核心取舍，同时让信息公开和成功率都更平滑。

## 新规则与概率表

1. 进入 `resolving_hand` 立即公开所有剩余牌的花色（点数始终保密、永不提前泄露，这条信息边界底线不变）。
2. 有赌博资格的判据不变：剩余恰为“1 张旧手牌 + 2 张新暗牌”共 3 张。具备资格时两个选项并存，不再有先后顺序或单向门：
   - 留牌：从三张中保留 1 张作为下回合手牌，其余 2 张回池。
   - 赌博：消耗全部 3 张牌，按花色组成走一次概率判定，不再保留下回合手牌。
3. 赌博成功率按 3 张牌的花色组成分档：

   | 花色组成 | 成功率 | 分类 |
   | --- | ---: | --- |
   | 3 张互不相同 | 10% | `mixed` |
   | 恰好 2 张同花色 | 35% | `twoMatching` |
   | 3 张全部同花色 | 75% | `allMatching` |

   以上数值以 `src/game/config/mahjong.ts` 的 `MAHJONG_HONOR_GAMBLE_SUCCESS_CHANCE` 常量为准，本记录不重复维护数值表。
4. 赌博成功后，中、發、白等概率（各 1/3）获得 1 张。无论成功还是失败，参与赌博的 3 张数牌全部回到摸牌池，下一回合都没有旧手牌——资源结算规则与旧版一致，只有成功率和信息公开时机改变。
5. 不具备赌博资格的情形（第一回合，或旧手牌已用于本轮建造，剩余 2 张全新牌）保持不变：公开花色后保留 1 张，其余回池，没有赌博入口。

## 实现要点

- 废除引擎动作 `revealMahjongHandSuits`；原先的 choosing/keeping 两种模式归并为单一布尔字段 `resolvingHand`——进入 `resolving_hand` 即视为花色已公开，不再需要一个额外动作把状态从“未公开”推进到“已公开”。
- 赌博结算函数改为两次独立随机：先按花色组成分类查概率表做成功判定；只有判定成功才进行第二次等概率随机，在中/發/白之间选取结果。失败路径不消耗第二次随机，避免多余的随机数调用，也让测试里对随机序列的断言更精确。
- `uiState` 新增 `honorGambleChance: number | null` 字段，供赌博按钮实时显示当前花色组成对应的成功率；不具备赌博资格或花色尚未公开时为 `null`。
- 结果提示文案改为中性措辞（“赌博成功”/“赌博未中”），不再使用暗示“花色不同就必然失败”的旧文案——因为现在即使三张全同花色（75%）也可能赌博失败。

## 不变式说明

- 108 张实体数牌的守恒不受影响：参与赌博的 3 张牌无论成败都返回摸牌池；中、發、白功能牌是 108 张实体之外的独立资源，不占用、不消耗实体 ID。
- 信息边界底线保持不变：点数在数牌于地图落地前，任何路径（UI、预览、无障碍文本、日志）都不得泄露。本次改动只是把“花色”公开的时机从“选择留牌分支之后才公开”提前到“进入 `resolving_hand` 立即公开”，点数保密规则完全未受影响。
- 手牌资源结算规则不变：走留牌分支下一回合最多 1 张旧手牌，走赌博分支下一回合没有旧手牌。

## 验证方式

- 新增/更新确定性随机序列单测矩阵：覆盖三种花色组成分类（`mixed`/`twoMatching`/`allMatching`）× 成功/失败两个分支 × 成功后中/發/白三种结果，断言判定读取的概率常量和随机调用顺序正确，并验证失败路径不触发第二次随机。
- 覆盖 `uiState.honorGambleChance` 在有/无赌博资格、花色公开前后的取值。
- 按仓库约定执行 `npm run lint`、`npm test`、`npm run build` 三者确认无回归。

## 后续事项

- 若后续为赌博结果增加动画或音效反馈，需要同步保持中性措辞与无障碍文本的一致性，避免重新引入“花色不同必然失败”式的误导文案。
